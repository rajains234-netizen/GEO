# Contributing to Geo

Thank you for your interest in contributing to the Geo project! We welcome contributions from the community and want to ensure a smooth process for everyone.

## Code of Conduct
We expect all contributors to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md). Please treat everyone with respect and courtesy.

## Development Setup
To set up the development environment:
1. Clone the repository:
   ```bash
   git clone https://github.com/rajains234-netizen/geo.git
   cd geo
   ```
2. Install dependencies:
   ```bash
   npm install
   ```

## Branching Strategy
- Use `feature/` for new features, e.g., `feature/login-page`.
- Use `bugfix/` for bug fixes, e.g., `bugfix/fix-typo`.
- Make sure to base your branch off `main`.

## Testing Guidelines
- Write tests for any new features or changes made.
- Run tests locally before submitting a pull request:
   ```bash
   npm test
   ```

## Pull Request Process
1. Ensure your branch is up to date with `main`.
2. Create your pull request using the GitHub interface.
3. Fill out the pull request template, detailing what was changed and why.
4. Request a review from at least one other contributor.
5. Implement any feedback received and update your pull request accordingly.

Thank you for contributing! We appreciate your help in improving the Geo project!
